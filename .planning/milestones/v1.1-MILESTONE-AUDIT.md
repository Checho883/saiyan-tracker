## Cross-Phase Integration Issues

### High Severity

**1. Sound preference not persisted (SET-01 / AUDIO-01)**
- `SoundProvider.tsx` line 30: `isMuted = useState(true)` — hardcoded, never reads `rewardStore.settings.sound_enabled`
- Settings page toggle persists to backend via `updateSettings({ sound_enabled })` but SoundProvider never consumes it
- BottomTabBar mute button operates only on in-memory state, also not persisted
- **Impact:** Sound always starts muted on every page load regardless of saved preference

**2. Stale transformation name after check (DASH-04)**
- `powerStore.updateFromCheck` updates `powerLevel` and `transformation` but not `transformationName`
- `ScouterHUD` displays stale form name until next `fetchPower()` call
- **Impact:** After a transformation event, scouter briefly shows wrong form name

### Medium Severity

**3. Capsule reveal chime missing (AUDIO-04)**
- `reveal_chime` sound ID defined in sprite map but `CapsuleDropOverlay.tsx` never calls `play('reveal_chime')` on tap
- Phase 6 SUMMARY explicitly noted Phase 7 should wire this; Phase 7 did not
- **Impact:** Capsule card-flip reveal is silent

### Low Severity

**4. Dead analytics API methods**
- `analyticsApi.capsuleHistory()` and `analyticsApi.wishHistory()` defined but never called
- Capsule/wish history views deferred to v1.2 (POLISH-05)
- **Impact:** None (dead code only)
