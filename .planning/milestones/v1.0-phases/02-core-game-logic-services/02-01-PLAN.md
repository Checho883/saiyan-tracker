  <behavior>
    - calc_daily_xp(1.0, 0, False) == 200 (100 * 1.0 * 2.0 * 1.0)
    - calc_daily_xp(0.8, 10, False) == 180 (100 * 0.8 * 1.5 * 1.5)

    - calc_daily_xp(0.3, 0, False) == 30 (100 * 0.3 * 1.0 * 1.0)
    - calc_daily_xp(1.0, 20, True) == 600 (100 * 1.0 * 2.0 * 2.0 * 1.5)
    - calc_daily_xp(0.0, 0, False) == 0 (zero completion = zero XP)
    - get_completion_tier(1.0) returns {"min_rate": 1.0, "multiplier": 2.0, "name": "kaio_x20", ...}
    - get_completion_tier(0.8) returns kaio_x10 tier (multiplier 1.5)
    - get_completion_tier(0.5) returns kaio_x3 tier (multiplier 1.2)
    - get_completion_tier(0.3) returns base tier (multiplier 1.0)
    - get_completion_tier(0.0) returns base tier
    - get_attribute_xp("normal") == 15
    - get_attribute_xp("important") == 22
    - get_attribute_xp("critical") == 30
    - calc_streak_bonus(0) == 0.0
    - calc_streak_bonus(10) == 0.5
    - calc_streak_bonus(20) == 1.0 (capped)
    - calc_streak_bonus(100) == 1.0 (still capped)
  </behavior>
  <action>
    1. Create `backend/app/services/__init__.py` as empty package init (will be populated with exports later).

    2. Create `backend/app/services/xp_service.py` with four pure functions:

    **calc_daily_xp(completion_rate: float, current_streak: int, zenkai_active: bool = False) -> int**
    - Formula: `math.floor(BASE_DAILY_XP * completion_rate * tier_multiplier * (1 + streak_bonus))`
    - If zenkai_active: multiply by `(1 + ZENKAI_BONUS)` before floor
    - Import constants from `app.core.constants`

    **get_completion_tier(completion_rate: float) -> dict**
    - Iterate COMPLETION_TIERS (already sorted desc by min_rate), return first match where `completion_rate >= tier["min_rate"]`
    - Fallback: return last tier (base) — though this should never trigger since base has min_rate=0.0

    **get_attribute_xp(importance: str) -> int**
    - Direct lookup from IMPORTANCE_XP dict
    - Raise KeyError for invalid importance (let it propagate — validation happens at API layer)

    **calc_streak_bonus(current_streak: int) -> float**
    - `min(current_streak * STREAK_BONUS_PER_DAY, STREAK_BONUS_CAP)`

    3. Create `backend/tests/test_xp_service.py` with tests covering all behavior items above. Use parametrized tests where inputs/outputs form a clear table. No DB needed — these are pure function tests.
  </action>
  <verify>
    <automated>cd backend && python -m pytest tests/test_xp_service.py -x -v</automated>
  </verify>
  <done>All four functions pass all test vectors. calc_daily_xp returns exact integers matching the formula for all bracket combinations including Zenkai. Tier lookup correctly selects the highest matching tier.</done>

    - get_xp_for_next_level(1) == 282 (100 * 2^1.5 = 282.84 -> 282)
  </behavior>
  <action>
    Create `backend/app/services/attribute_service.py` with three pure functions:

    **calc_attribute_level(total_xp: int) -> int**
    - Iteratively sum XP thresholds: level N needs `floor(ATTRIBUTE_LEVEL_BASE_XP * N^ATTRIBUTE_LEVEL_FORMULA_EXPONENT)` XP
    - Keep incrementing level while cumulative XP consumed <= total_xp
    - Return highest achieved level

    **get_attribute_title(attribute: str, level: int) -> str | None**
    - Look up ATTRIBUTE_TITLES[attribute], iterate thresholds in ascending order
    - Return the title for the highest threshold <= level, or None if below all thresholds

    **get_xp_for_next_level(current_level: int) -> int**
    - Returns `floor(ATTRIBUTE_LEVEL_BASE_XP * (current_level + 1)^ATTRIBUTE_LEVEL_FORMULA_EXPONENT)`
    - This is the XP cost for the NEXT level (not cumulative)

    Create `backend/tests/test_attribute_service.py` with tests covering all behavior items. Use parametrize for the title threshold tests.
  </action>

    <automated>cd backend && python -m pytest tests/test_attribute_service.py -x -v</automated>
  </verify>
  <done>Attribute level calculation produces correct level for cumulative XP inputs. Title lookup returns the correct title at each threshold boundary. XP-for-next-level returns exact integer costs.</done>
