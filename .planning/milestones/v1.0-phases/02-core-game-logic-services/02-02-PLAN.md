    - roll_capsule_drop returns None when random > 0.25 (no drop)
    - roll_capsule_drop returns CapsuleDrop with reward when random <= 0.25
    - select_rarity_tier picks from weighted distribution (common 60%, rare 30%, epic 10%)
    - If rolled rarity has no rewards, falls back: epic -> rare -> common
    - If no rewards exist at all, returns None (no drop even if RNG said yes)
    - Capsule system returns None immediately when no active rewards exist in DB

    Dragon Ball service:
    - award_dragon_ball increments User.dragon_balls_collected by 1
    - award_dragon_ball returns True when dragon_balls_collected reaches 7 (wish available)
    - revoke_dragon_ball decrements User.dragon_balls_collected by 1 (for uncheck clawback)
    - grant_wish resets dragon_balls_collected to 0, increments wishes_granted, increments wish.times_wished, creates WishLog
    - grant_wish raises error if dragon_balls_collected < 7

    Power service:
    - get_transformation_for_power(0) returns "base"
    - get_transformation_for_power(1000) returns "ssj"
    - get_transformation_for_power(150000) returns "beast"
    - get_transformation_for_power(999) returns "base" (just below threshold)
    - check_transformation_change("base", 1000) returns {"key": "ssj", "name": "Super Saiyan", "threshold": 1000}
    - check_transformation_change("ssj", 1500) returns None (no change)
    - recalculate_power_level sums all DailyLog.xp_earned for user and updates User.power_level
  </behavior>
  <action>
    1. **Extend conftest.py** with new fixtures needed for DB-touching tests:
    - `sample_habit(db, sample_user)`: Creates a Habit with user_id, title="Test Habit", attribute="str", importance="normal", start_date="2026-01-01"
    - `sample_reward(db, sample_user)`: Creates a Reward with user_id, title="Test Reward", rarity="common", is_active=True
    - `sample_wish(db, sample_user)`: Creates a Wish with user_id, title="Test Wish", is_active=True

    2. **Create `backend/app/services/capsule_service.py`:**

    **select_rarity_tier() -> str**
    - Uses `random.choices(list(CAPSULE_RARITY_WEIGHTS.keys()), weights=list(CAPSULE_RARITY_WEIGHTS.values()), k=1)[0]`

    **roll_capsule_drop(db: Session, user_id: UUID, habit_id: UUID) -> CapsuleDrop | None**
    - Check if any active rewards exist: `db.query(Reward).filter(Reward.user_id == user_id, Reward.is_active == True).count()`. If 0, return None.
    - Roll: `if random.random() > CAPSULE_DROP_CHANCE: return None`
    - Pick rarity with `select_rarity_tier()`
    - Fallback order: RARITY_FALLBACK_ORDER = ["epic", "rare", "common"]. Start from rolled rarity's index, iterate downward.
    - For each tier, query rewards. If found, pick `random.choice(rewards)`, create and return CapsuleDrop (do NOT add to session — caller manages session).
    - If no tier has rewards, return None.
    - IMPORTANT: Do not call db.add() or db.flush() — the caller (check_habit) manages the session.

    3. **Create `backend/app/services/dragon_ball_service.py`:**

    **award_dragon_ball(user: User) -> dict**
    - `user.dragon_balls_collected += 1`
    - Returns `{"dragon_balls_collected": user.dragon_balls_collected, "wish_available": user.dragon_balls_collected >= DRAGON_BALLS_REQUIRED}`
    - Does NOT commit — caller manages session.

    **revoke_dragon_ball(user: User) -> None**
    - `user.dragon_balls_collected = max(0, user.dragon_balls_collected - 1)`

    **grant_wish(db: Session, user: User, wish_id: UUID) -> dict**
    - Verify `user.dragon_balls_collected >= DRAGON_BALLS_REQUIRED`, raise ValueError if not
    - Look up Wish by wish_id, raise ValueError if not found or not active
    - Reset `user.dragon_balls_collected = 0`
    - Increment `user.wishes_granted += 1`
    - Increment `wish.times_wished += 1`
    - Create WishLog(user_id=user.id, wish_id=wish.id)
    - `db.add(wish_log)` and `db.flush()` (within caller's transaction)
    - Return `{"wish_title": wish.title, "times_wished": wish.times_wished, "wishes_granted": user.wishes_granted}`

    4. **Create `backend/app/services/power_service.py`:**

    **get_transformation_for_power(power_level: int) -> dict** (pure function)
    - Iterate TRANSFORMATIONS, return last one where `power_level >= threshold`

    **check_transformation_change(old_transformation: str, new_power_level: int) -> dict | None** (pure function)
    - Get new transformation via `get_transformation_for_power(new_power_level)`
    - If `new_transform["key"] != old_transformation`, return the new transform dict
    - Else return None

    **recalculate_power_level(db: Session, user: User) -> int**
    - `total = db.query(func.coalesce(func.sum(DailyLog.xp_earned), 0)).filter(DailyLog.user_id == user.id).scalar()`
    - `user.power_level = total`
    - Return total

    5. **Create test files:** `test_capsule_service.py`, `test_dragon_ball_service.py`, `test_power_service.py`
    - Capsule tests: Use `unittest.mock.patch("random.random")` and `patch("random.choices")` to control RNG. Test the fallback order. Test no-rewards scenario.
    - Dragon Ball tests: Use db fixture with sample_user. Test increment, threshold detection, revoke, and wish granting flow.
    - Power tests: Pure function tests for transformation lookup + DB test for recalculate_power_level using sample DailyLog entries.
