  <action>
Modify HeroSection.tsx to render two layout variants using Tailwind responsive classes:

**Mobile layout (default, below md:768px):** A compact single-row layout:
- Horizontal flex row with items-center
- Left: AuraGauge at size={48} with SaiyanAvatar at w-6 h-6 overlaid
- Center: Vertical stack with power level (text-lg font-mono font-bold text-saiyan-500), transformation name (text-xs text-text-secondary truncate), and inline aura progress bar (h-1.5 rounded-full)
- The tier change banner (TierChangeBanner) should still render when present — position it absolutely above the compact hero or let it render inline below

**Desktop layout (md: and above):** The current full vertical layout (unchanged).

Implementation approach — use Tailwind `hidden`/`md:block` and `md:hidden` to toggle between layouts:

```tsx
export function HeroSection() {
  // ... existing hooks unchanged ...

  return (
    <>
      {/* === COMPACT HERO (mobile < 768px) === */}
      <div className="md:hidden flex items-center gap-3 px-4 py-3 bg-gradient-to-b from-space-800 to-transparent">
        {/* Small avatar + gauge */}
        <div className="relative flex-shrink-0">
          <AuraGauge percent={percent} tier={tier} size={48} />
          <div className="absolute inset-0 flex items-center justify-center">
            <SaiyanAvatar transformation={transformation} className="w-6 h-6" />
          </div>
        </div>

        {/* Power info inline */}
        <div className="flex-1 min-w-0">
          <div className="flex items-baseline gap-2">
            <span className="text-lg font-mono font-bold text-saiyan-500">
              {powerLevel.toLocaleString()}
            </span>
            <span className="text-xs text-text-secondary truncate">
              {transformationName}
            </span>
          </div>
          {/* Inline progress bar showing aura percent */}
          <div className="h-1.5 bg-space-700 rounded-full mt-1 overflow-hidden">
            <div
              className={`h-full rounded-full transition-all duration-500`}
              style={{
                width: `${percent}%`,
                backgroundColor: strokeColorForTier(tier),
              }}
            />
          </div>
        </div>

        {/* Display name (if set) */}
        {displayName && (
          <span className="text-sm font-bold text-text-primary truncate max-w-[80px]">
            {displayName}
          </span>
        )}
      </div>

      {/* === FULL HERO (desktop md+) === */}
      <div className="hidden md:flex flex-col items-center gap-4 px-4 py-6 bg-gradient-to-b from-space-800 to-transparent">
        {/* ... existing full hero layout unchanged ... */}
        <div className="relative">
          <AuraGauge percent={percent} tier={tier} size={160} />
          <div className="absolute inset-0 flex items-center justify-center">
            <SaiyanAvatar transformation={transformation} className="w-20 h-20" />
          </div>
          {tierEvent && tierEvent.type === 'tier_change' && (
            <TierChangeBanner tier={tierEvent.tier} onDismiss={handleTierDismiss} />
          )}
        </div>
        {displayName && (
          <div className="bg-space-700/50 rounded-lg px-3 py-1">
            <span className="text-lg font-bold text-text-primary">{displayName}</span>
          </div>
        )}
        <ScouterHUD
          powerLevel={powerLevel}
          transformationName={transformationName}
          nextTransformation={nextTransformation}
          nextThreshold={nextThreshold}
        />
      </div>

      {/* Tier change banner for mobile — render below compact hero */}
      {tierEvent && tierEvent.type === 'tier_change' && (
        <div className="md:hidden">
          <TierChangeBanner tier={tierEvent.tier} onDismiss={handleTierDismiss} />
        </div>
      )}
    </>
  );
}
```

For the inline progress bar color in compact hero, you need to map tier to CSS color. Create a helper or use the same color map pattern from AuraGauge.tsx (strokeColorMap). You can extract the color mapping to a shared constant or duplicate the small map inline.

Also verify AuraGauge works at size={48}: The SVG uses viewBox based on size, so it should scale correctly. The center text (fontSize="22") and tier label (fontSize="10") will be too large at 48px. For the compact hero, the percent text and tier label inside the gauge are NOT needed since power level is shown inline. Two options:
- Pass a `hideText` prop to AuraGauge, OR
- Simply render a simpler version (just the circular progress ring) in compact mode

Recommended: Add an optional `hideText?: boolean` prop to AuraGauge. When true, skip rendering the `<text>` elements. Default false to not break existing usage.

```tsx
</NavLink>
```

```tsx
<button
  onClick={toggleMute}

  <SoundIcon size={24} />
  <span className="text-xs">Sound</span>
</button>
```
