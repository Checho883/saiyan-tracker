2. Add `GET /analytics/off-day-summary` endpoint to `analytics.py`:
   - Accept `period: Literal["week", "month", "all"]` param (same as /summary)
   - Query OffDay records filtered by period
   - Calculate `total_off_days` = count of off-days in period
   - Calculate `xp_impact_estimate`:

     - Multiply avg_daily_xp by total_off_days
   - Calculate `streaks_preserved`:
     - For each off-day in period, check if the user's overall Streak.last_active_date was within 1 day before the off-day (i.e., streak was active)
     - Simpler approach: count off-days where there exists a DailyLog on the day before OR the day after the off-day (meaning the streak bridged over the off-day)
   - Calculate `reason_breakdown`: GROUP BY OffDay.reason with COUNT
   - Import OffDay model and add to existing imports

    - Test: monthly_rate and monthly_delta use 30-day rolling windows

1. Add Pydantic schema `CompletionTrend` to `backend/app/schemas/analytics.py`:
   ```python
   class CompletionTrend(BaseModel):
       weekly_rate: float          # completion rate for last 7 days
       weekly_delta: float         # current - previous week (percentage points)
       weekly_habits_due: int      # raw count for last 7 days
       weekly_habits_completed: int
       monthly_rate: float         # completion rate for last 30 days
       monthly_delta: float        # current - previous month (percentage points)
       monthly_habits_due: int     # raw count for last 30 days
       monthly_habits_completed: int
   ```

2. Add `GET /analytics/completion-trend` endpoint to `analytics.py`:
   - No period param needed — always returns both weekly and monthly
   - For weekly: query DailyLogs from last 7 days, sum habits_due and habits_completed
   - For previous week: query DailyLogs from 14-7 days ago
   - weekly_rate = weekly_completed / weekly_due (0.0 if due=0)
   - weekly_delta = weekly_rate - prev_weekly_rate (percentage point difference)
   - Same pattern for monthly (30 days, previous 30 days = 60-30 days ago)
   - Uses snapshotted DailyLog.habits_due as denominator (per success criteria)
