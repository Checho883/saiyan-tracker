    - Test: SoundProvider stays muted when settings.sound_enabled is false or settings is null

    - Test: No infinite re-render loop (useEffect only fires when sound_enabled changes)

  <action>
**In SoundProvider.tsx:**

1. Add import at top:
```typescript
import { useRewardStore } from '../store/rewardStore';
```

2. Inside `SoundProvider` function, after the `useState(true)` line, add a Zustand selector:
```typescript
const soundEnabled = useRewardStore((s) => s.settings?.sound_enabled);
```

3. Add a `useEffect` AFTER the existing cleanup `useEffect` to sync mute state from settings:
```typescript
useEffect(() => {
  if (soundEnabled !== undefined) {
    const shouldMute = !soundEnabled;
    setIsMuted(shouldMute);
    Howler.mute(shouldMute);
  }
}, [soundEnabled]);
```

4. Update `toggleMute` to persist preference to backend:
```typescript
const toggleMute = useCallback(() => {
  setIsMuted((prev) => {
    const newMuted = !prev;
    Howler.mute(newMuted);
    useRewardStore.getState().updateSettings({ sound_enabled: !newMuted });
    return newMuted;
  });
}, []);
```

**In sound-provider.test.tsx:**

Add test cases covering the settings sync behavior. Mock `useRewardStore` to return controlled `settings.sound_enabled` values. Verify that:
- When `sound_enabled` is `true`, `isMuted` becomes `false`
- When `sound_enabled` is `undefined` (settings not loaded yet), `isMuted` stays `true` (default)
- When `toggleMute` is called, `updateSettings` is invoked with the new sound_enabled value

IMPORTANT: Use `vi.mock('../store/rewardStore')` for the rewardStore mock. The existing test file already mocks howler — keep that mock intact.

**Anti-pattern to avoid:**
- Do NOT add `settings` (the full object) as a useEffect dependency — use only `soundEnabled` (the boolean) to prevent re-renders on unrelated settings changes.
- Do NOT create an infinite loop: the useEffect reads from Zustand, and toggleMute writes to Zustand. The guard `soundEnabled !== undefined` and the fact that setting the same value is a no-op in React state prevents cycles.
