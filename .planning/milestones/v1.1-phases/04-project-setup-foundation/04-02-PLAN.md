  <action>
    Create all four stores using `create` from `zustand`. Each store follows the pattern from 04-RESEARCH.md Pattern 3.

    **CRITICAL -- Toast error handling (locked decision):**
    Every store catch block that handles API errors MUST call `toast.error()` from `react-hot-toast`:
    ```typescript
    import toast from 'react-hot-toast';
    // In catch blocks:
    catch (err) {
      const message = (err as Error).message;
      toast.error(message, { duration: 4000 });
      set({ error: message, isLoading: false });
    }
    ```
    This implements the locked CONTEXT.md decision: "Toast notifications for API/network errors -- non-blocking, auto-dismiss after 3-5s".

    **habitStore.ts:**
    - State: `todayHabits: HabitTodayResponse[]`, `isLoading: boolean`, `error: string | null`
    - Actions:
      - `fetchToday(date: string)` -- calls `habitsApi.todayList(date)`, sets todayHabits. Catch block: `toast.error(message, { duration: 4000 })`.
      - `checkHabit(habitId: string, date: string)` -- optimistic toggle: flip `completed` immediately, call `habitsApi.check()`, on success update from server response AND distribute to other stores:
        - Call `usePowerStore.getState().updateFromCheck(result.power_level, result.transformation)` to update power
        - Enqueue animations via `useUiStore.getState().enqueueAnimation()` for: transform_change, capsule drop, is_perfect_day, xp_popup (attribute_xp_awarded > 0), dragon_ball (dragon_ball?.dragon_balls_collected changes), tier change (daily_log.completion_tier changes)
        - On error: rollback todayHabits to previous state, set error, `toast.error(message, { duration: 4000 })`
      - Return the `CheckHabitResponse` so components can access quote data
    - Add comment showing `useShallow` usage pattern for components

    **powerStore.ts:**
    - State: `powerLevel: number`, `transformation: string`, `transformationName: string`, `nextTransformation: string | null`, `nextThreshold: number | null`, `dragonBallsCollected: number`, `wishesGranted: number`, `attributes: AttributeDetail[]`, `isLoading: boolean`, `error: string | null`
    - Actions:
      - `fetchPower()` -- calls `powerApi.current()`, sets all fields. Catch block: `toast.error(message, { duration: 4000 })`.
      - `updateFromCheck(powerLevel: number, transformation: string)` -- lightweight update from habit check response (avoids full re-fetch)
    - Add comment showing `useShallow` usage pattern

    **rewardStore.ts:**
    - State: `rewards: RewardResponse[]`, `wishes: WishResponse[]`, `categories: CategoryResponse[]`, `settings: SettingsResponse | null`, `isLoading: boolean`, `error: string | null`
    - Actions:
      - `fetchRewards()` -- calls `rewardsApi.list()`, sets rewards. Catch block: `toast.error(message, { duration: 4000 })`.
      - `fetchWishes()` -- calls `wishesApi.list()`, sets wishes. Catch block: `toast.error(message, { duration: 4000 })`.
      - `fetchCategories()` -- calls `categoriesApi.list()`, sets categories. Catch block: `toast.error(message, { duration: 4000 })`.
      - `fetchSettings()` -- calls `settingsApi.get()`, sets settings. Catch block: `toast.error(message, { duration: 4000 })`.
      - CRUD stubs for rewards, wishes, categories (create/update/delete that call API + update local state). All catch blocks: `toast.error(message, { duration: 4000 })`.
      - `updateSettings(data: SettingsUpdate)` -- calls `settingsApi.update()`, sets settings. Catch block: `toast.error(message, { duration: 4000 })`.
    - Add comment showing `useShallow` usage pattern

    **uiStore.ts:**
    - State: `animationQueue: AnimationEvent[]`, `activeModal: string | null`
    - AnimationEvent type (union type): `tier_change`, `perfect_day`, `capsule_drop`, `dragon_ball`, `transformation`, `xp_popup`, `shenron` -- exactly as in 04-RESEARCH.md Pattern 6
    - Actions:
      - `enqueueAnimation(event: AnimationEvent)` -- append to queue
      - `dequeueAnimation()` -- shift first, return it
      - `clearAnimations()` -- empty queue
      - `openModal(id: string)` -- set activeModal
      - `closeModal()` -- set activeModal to null

    **useInitApp.ts:**
    - Hook that runs on mount (useEffect with empty deps)
    - Calls `Promise.all()` with ALL of these (CRITICAL -- do not omit any):
      - `useHabitStore.getState().fetchToday(today)` where today is `new Date().toISOString().split('T')[0]`
      - `usePowerStore.getState().fetchPower()`
      - `useRewardStore.getState().fetchRewards()`
      - `useRewardStore.getState().fetchWishes()`
      - `useRewardStore.getState().fetchCategories()`
      - `useRewardStore.getState().fetchSettings()`
    - Returns `{ isReady: boolean, error: string | null }`
    - Sets isReady to true after all resolve, error on catch
