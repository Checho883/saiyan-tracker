  <action>
    1. Run `npm create vite@latest frontend -- --template react-ts` from project root to scaffold the Vite 7 + React 19 + TypeScript project.

    2. From frontend/, install all dependencies:
       ```
       npm install zustand ky react-router react-hot-toast lucide-react
       npm install -D @tailwindcss/vite tailwindcss
       ```

    3. Update `vite.config.ts` to use the Tailwind CSS Vite plugin (NOT PostCSS):
       ```typescript
       import { defineConfig } from 'vite';
       import react from '@vitejs/plugin-react';
       import tailwindcss from '@tailwindcss/vite';

       export default defineConfig({
         plugins: [react(), tailwindcss()],
         server: {
           port: 5173,
           proxy: {
             '/api': {
               target: 'http://localhost:8000',
               changeOrigin: true,
             },
           },
         },
       });
       ```

    4. Replace `src/index.css` with Tailwind import and @theme color tokens. Use the exact color system from 04-RESEARCH.md Pattern 1:
       - `@import "tailwindcss";` at the top
       - `@theme { }` block with all color tokens:
         - Background layers: space-900 (#0a0a0f), space-800 (#0d0d1a), space-700 (#141428), space-600 (#1a1a35)
         - Orange accent (Goku gi): saiyan-500 (#ff6b00), saiyan-400 (#ff8533), saiyan-600 (#cc5500)
         - Blue accent (SSGSS aura): aura-500 (#1e90ff), aura-400 (#4da6ff), aura-600 (#0070dd)
         - Text: text-primary (#e8e8f0), text-secondary (#9898b0), text-muted (#5a5a78)
         - Semantic: success (#22c55e), danger (#ef4444), warning (#f59e0b)
         - Glow effects: glow-orange (oklch(0.7 0.2 50 / 0.3)), glow-blue (oklch(0.6 0.2 250 / 0.3))
         - Attribute colors for RPG bars: attr-str (#ef4444 red), attr-vit (#22c55e green), attr-int (#3b82f6 blue), attr-ki (#f59e0b amber)
         - Rarity colors: rarity-common (#9898b0), rarity-rare (#1e90ff), rarity-epic (#a855f7 purple)
       - Global body styles: `body { @apply bg-space-900 text-text-primary; }`

    5. Update `src/main.tsx` to be minimal -- just ReactDOM.createRoot rendering a placeholder App:
       ```typescript
       import { StrictMode } from 'react';
       import { createRoot } from 'react-dom/client';
       import './index.css';

       createRoot(document.getElementById('root')!).render(
         <StrictMode>
           <div className="min-h-screen bg-space-900 text-text-primary flex items-center justify-center">
             <p className="text-saiyan-500 text-2xl">Saiyan Tracker</p>
           </div>
         </StrictMode>,
       );
       ```
       This placeholder will be replaced in Plan 02 when App.tsx is wired.

    6. Delete any scaffold files not needed: `src/App.css`, `src/App.tsx` (will be recreated in Plan 02), `src/assets/react.svg`, `public/vite.svg`.

    7. Do NOT create `tailwind.config.js` or `postcss.config.js` -- Tailwind v4 uses CSS-first config via @theme.

    8. Ensure `tsconfig.app.json` has `"strict": true` for type safety.

    2. Create `src/services/api.ts` with:
       - `API_BASE` from `import.meta.env.VITE_API_BASE ?? 'http://localhost:8000/api/v1'`
       - Single `api` ky instance via `ky.extend()` with:
         - `prefixUrl: API_BASE` (no trailing slash)
         - `retry: { limit: 2, methods: ['get'] }`
         - `afterResponse` hook that throws Error with `detail` from response body on non-OK responses
       - Typed endpoint objects exactly as documented in 04-RESEARCH.md Pattern 2:
