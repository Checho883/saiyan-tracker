<task type="auto">
  <name>Task 1: Create vercel.json and frontend .env.example</name>
  <files>frontend/vercel.json, frontend/.env.example</files>
  <action>
1. Create `frontend/vercel.json` with SPA rewrite rule. This file must be in `frontend/` because that directory is set as the Vercel Root Directory:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

This ensures that direct URL access to any React Router route (e.g., /analytics, /settings) returns the SPA shell instead of a 404.

2. Create `frontend/.env.example` documenting the VITE_API_BASE variable:

```env
# Saiyan Tracker Frontend Configuration
# Copy to .env.local for local development (optional — dev uses Vite proxy)
# Set VITE_API_BASE in Vercel dashboard for production builds

# API Base URL — production: your VPS address
# VITE_API_BASE=http://<VPS_IP>:8000/api/v1
```

Note: The frontend already reads VITE_API_BASE correctly in `src/services/api.ts` with a localhost fallback. No code changes needed for DEPLOY-06 — just Vercel dashboard configuration documented in .env.example.
  </action>
  <verify>

  </verify>
  <done>vercel.json exists in frontend/ with SPA rewrite rule routing all paths to index.html. frontend/.env.example documents VITE_API_BASE.</done>
