<objective>
Refactor the animation queue from FIFO to a priority-tiered system with combo batching.

Purpose: Prevent animation overload when multiple events fire simultaneously (e.g., a single habit check triggers transformation + capsule_drop + perfect_day + dragon_ball + xp_popup + tier_change). Events must play in priority order and batch into a combo summary when 3+ queue up.

Output: Priority-aware uiStore queue, refactored AnimationPlayer with combo logic, new ComboSummaryOverlay component, updated tests.
</objective>
