<objective>
Enable SQLite foreign key enforcement on the file-based database and clean up any orphaned rows from past operations without FK enforcement.

Purpose: The file-based SQLite database currently has foreign keys silently disabled due to a conditional guard that only activates for in-memory databases. This means deletes could have left orphaned child rows. Fixing this is the prerequisite for all subsequent data integrity work.

Output: session.py with unconditional FK enforcement, a cleanup script that detects and removes orphaned rows
</objective>

Remove the `if ":memory:" in DATABASE_URL:` guard around the PRAGMA foreign_keys event listener. The listener should fire unconditionally for all SQLite connections. The fixed code should be:

```python
from sqlalchemy import create_engine, event
from sqlalchemy.orm import sessionmaker
from app.core.config import DATABASE_URL

connect_args = {}
if "sqlite" in DATABASE_URL:
    connect_args = {"check_same_thread": False}

engine = create_engine(DATABASE_URL, connect_args=connect_args)

# Enable foreign key enforcement for SQLite (required on every connection)
@event.listens_for(engine, "connect")
def set_sqlite_pragma(dbapi_conn, connection_record):
    cursor = dbapi_conn.cursor()
    cursor.execute("PRAGMA foreign_keys=ON")
    cursor.close()

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

Remove the misleading comment about "in-memory SQLite" and "WAL mode". The PRAGMA is for FK enforcement, not WAL.

<task type="auto">
  <name>Task 2: Create integrity check and orphan cleanup script</name>
  <files>backend/scripts/check_db_integrity.py</files>
  <action>
Create `backend/scripts/check_db_integrity.py` -- a standalone script that:

1. Imports the engine from `app.database.session` (so FK enforcement is already active)
2. Runs `PRAGMA foreign_key_check` and prints results
3. For each known parent-child relationship, runs a LEFT JOIN query to find orphaned rows:
   - habit_logs where habit_id not in habits
   - habit_streaks where habit_id not in habits
   - task_completions where task_id not in tasks
   - tasks where category_id not in task_categories
   - habits where category_id not in task_categories
4. Reports counts of orphaned rows found
5. If `--fix` flag is passed, deletes orphaned rows and reports what was deleted
6. If no `--fix` flag, prints "Run with --fix to clean up" when orphans are found
7. Exits with code 0 if no orphans, code 1 if orphans found (without --fix)

Use raw SQL via `engine.connect()` and `connection.execute(text(...))`. Do NOT import models or use ORM -- this is a database-level check.

Include a `if __name__ == "__main__":` block with argparse for the `--fix` flag.

The script should be runnable as: `cd backend && python -m scripts.check_db_integrity` or `cd backend && python scripts/check_db_integrity.py`

For the second form to work, add sys.path manipulation at the top:
```python
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).resolve().parent.parent))
```
