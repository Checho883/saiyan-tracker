**CollapsibleSection.tsx — Responsive default state:**
The user wants sections collapsed by default on mobile, open by default on desktop. Since this is a React component using useState, we need to detect the initial viewport width. Use a simple window.innerWidth check for the initial state only (no ongoing listener needed):

```tsx
export function CollapsibleSection({
  title,
  icon: Icon,
  defaultOpen = false,
  count,
  children,
}: CollapsibleSectionProps) {
  const [isOpen, setIsOpen] = useState(() => {
    // On mobile (< 768px), override to collapsed unless explicitly opened
    if (typeof window !== 'undefined' && window.innerWidth < 768) {
      return false;
    }
    return defaultOpen;
  });
  // ... rest unchanged
```

This means on mobile, ALL sections start collapsed regardless of the `defaultOpen` prop. On desktop, they respect the prop (Preferences starts open, others closed — current behavior).
