**1. deploy/saiyan-tracker.service** — systemd unit file:
```ini
[Unit]
Description=Saiyan Tracker FastAPI Backend
After=network.target

[Service]
Type=simple
User=saiyan
Group=saiyan
WorkingDirectory=/home/user/saiyan-tracker/backend
EnvironmentFile=/etc/saiyan-tracker.env
ExecStart=/home/user/saiyan-tracker/venv/bin/uvicorn app.main:app --uds /run/saiyan-tracker.sock
Restart=on-failure
RestartSec=5
UMask=0117

# Socket directory
RuntimeDirectory=saiyan-tracker

[Install]
WantedBy=multi-user.target
```

Key details:
- `UMask=0117` creates socket as `srw-rw----` (owner + group can access)
- `RuntimeDirectory=saiyan-tracker` auto-creates `/run/saiyan-tracker/` — BUT we use `--uds /run/saiyan-tracker.sock` (at `/run/` level, not inside a subdirectory). Remove `RuntimeDirectory` and instead note in SETUP.md that `www-data` must be added to the `saiyan` group. The socket at `/run/saiyan-tracker.sock` will be created by uvicorn.
- Actually, to avoid needing `/run/` write access, use `RuntimeDirectory=saiyan-tracker` and set `--uds /run/saiyan-tracker/saiyan-tracker.sock`. Update the Nginx upstream to match.

Final socket path: `/run/saiyan-tracker/saiyan-tracker.sock`
- `RuntimeDirectory=saiyan-tracker` creates `/run/saiyan-tracker/` owned by `saiyan:saiyan`
- uvicorn `--uds /run/saiyan-tracker/saiyan-tracker.sock`
- Nginx upstream: `unix:/run/saiyan-tracker/saiyan-tracker.sock`

**2. deploy/saiyan-tracker.nginx** — Nginx server block:
```nginx
upstream saiyan_backend {
    server unix:/run/saiyan-tracker/saiyan-tracker.sock;
}

server {
    listen 80;
    server_name api.saiyantracker.com;

    location / {
        proxy_pass http://saiyan_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_connect_timeout 10s;
    }
}
```

Certbot will auto-modify this to add SSL. Do NOT add SSL directives manually.

**1. deploy/deploy.sh** — deployment automation script:
```bash
#!/usr/bin/env bash
set -euo pipefail

PROJECT_DIR="/home/user/saiyan-tracker"
BACKEND_DIR="${PROJECT_DIR}/backend"
DB_FILE="${BACKEND_DIR}/saiyan_tracker.db"

echo "=== Saiyan Tracker Deploy ==="

# Backup database
if [ -f "$DB_FILE" ]; then
    cp "$DB_FILE" "${DB_FILE}.bak"
    echo "Database backed up to ${DB_FILE}.bak"
fi

# Pull latest code
cd "$PROJECT_DIR"
git pull origin main
echo "Code updated"

# Install Python dependencies
source "${PROJECT_DIR}/venv/bin/activate"
pip install -r "${BACKEND_DIR}/requirements.txt" --quiet
echo "Dependencies installed"

# Fix DB file ownership (service runs as 'saiyan' user)
sudo chown saiyan:saiyan "$DB_FILE" "${DB_FILE}-shm" "${DB_FILE}-wal" 2>/dev/null || true
echo "DB ownership fixed"

# Restart service
sudo systemctl restart saiyan-tracker
echo "Service restarted"

# Quick health check
sleep 2
if curl -sf http://localhost/health > /dev/null 2>&1; then
    echo "Health check: OK"
else
    echo "Health check: FAILED — check 'sudo journalctl -u saiyan-tracker -n 20'"
fi

echo "=== Deploy complete ==="
```

Make the script executable: `chmod +x deploy/deploy.sh`

**2. deploy/smoke-test.sh** — post-deploy verification:
```bash
#!/usr/bin/env bash
echo "=== Saiyan Tracker Smoke Test ==="

echo "1. Service status:"
systemctl is-active saiyan-tracker && echo "   PASS" || echo "   FAIL"

echo "2. HTTPS health check:"
if curl -sf https://api.saiyantracker.com/health | grep -q '"ok"'; then
    echo "   PASS"
else
    echo "   FAIL"
fi

echo "3. WAL mode:"
WAL=$(sqlite3 /home/user/saiyan-tracker/backend/saiyan_tracker.db "PRAGMA journal_mode;" 2>/dev/null)
if [ "$WAL" = "wal" ]; then
    echo "   PASS (journal_mode=$WAL)"
else
    echo "   FAIL (journal_mode=$WAL)"
fi

echo "4. Env file permissions:"
PERMS=$(stat -c '%a' /etc/saiyan-tracker.env 2>/dev/null)
if [ "$PERMS" = "640" ]; then
    echo "   PASS (permissions=$PERMS)"
else
    echo "   FAIL (permissions=$PERMS)"
fi

echo "5. Socket exists:"
if [ -S /run/saiyan-tracker/saiyan-tracker.sock ]; then
    echo "   PASS"
else
    echo "   FAIL"
fi

echo "=== Done ==="
```

Make the script executable: `chmod +x deploy/smoke-test.sh`

**3. deploy/SETUP.md** — step-by-step VPS setup guide:

Write a concise guide covering these steps IN ORDER:

1. **Prerequisites** — SSH access, DNS A record for api.saiyantracker.com pointing to VPS IP
2. **System setup** — Install nginx, certbot, python3-venv, sqlite3. Create system user: `sudo useradd --system --no-create-home --shell /usr/sbin/nologin saiyan`. Add www-data to saiyan group: `sudo usermod -aG saiyan www-data`
3. **Clone repo** — `git clone` to `/home/user/saiyan-tracker`, create venv, install requirements
4. **Environment file** — Copy `deploy/saiyan-tracker.env.example` to `/etc/saiyan-tracker.env`, fill in values, `chmod 640`, `chown root:saiyan`
5. **systemd service** — Copy `deploy/saiyan-tracker.service` to `/etc/systemd/system/`, `systemctl daemon-reload`, `systemctl enable --now saiyan-tracker`
6. **Nginx** — Copy `deploy/saiyan-tracker.nginx` to `/etc/nginx/sites-available/saiyan-tracker`, symlink to sites-enabled, remove default, `nginx -t && systemctl reload nginx`
7. **TLS certificate** — `sudo certbot --nginx -d api.saiyantracker.com`
8. **Verify** — Run `deploy/smoke-test.sh`
9. **Future deploys** — Run `deploy/deploy.sh`

Keep the guide practical — exact commands, no theory. Each step should be copy-pasteable.
